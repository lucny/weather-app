# Inicializace Vite + React projektu - Weather App

## Příkaz inicializace
```bash
npm create vite@latest weather-app -- --template react
```

## Výsledná struktura projektu

### Hlavní adresář
```
weather-app/
├── docs/                    # Dokumentace projektu
├── node_modules/            # Nainstalované balíčky
├── public/                  # Statické soubory (favikony, obrázky)
├── src/                     # Zdrojový kód React aplikace
│   ├── assets/             # Obrázky a média
│   ├── App.jsx             # Hlavní React komponenta
│   ├── App.css             # Styly pro App komponentu
│   ├── index.css           # Globální CSS styly
│   └── main.jsx            # Entry point aplikace
├── index.html              # HTML template
├── package.json            # Konfigurace projektu a závislosti
├── package-lock.json       # Lock soubor pro konzistenci verzí
├── vite.config.js          # Konfigurace Vite build toolů
├── eslint.config.js        # Konfigurace linteru
├── README.md               # Readme soubor
└── .gitignore              # Ignorované soubory pro Git
```

## Co se vytvoří po inicializaci?

### 1. **package.json** - Konfigurační soubor Node.js projektu
- **name**: Jméno balíčku (`weather-app`)
- **version**: Počáteční verze (`0.0.0`)
- **type**: `"module"` - projekt používá ES6 moduly
- **scripts**:
  - `npm run dev` - Spustí vývojový server s hot reload
  - `npm run build` - Vytvoří produkční build
  - `npm run lint` - Spustí ESLint kontrolu kódu
  - `npm run preview` - Náhled produkčního buildu

### 2. **Hlavní dependence**
#### Production dependencies (runtime):
- **react** (^19.2.6) - Knihovna pro tvorbu uživatelských rozhraní
- **react-dom** (^19.2.6) - Rendering React do DOM

#### Development dependencies (pouze pro vývoj):
- **vite** (^8.0.12) - Build tool a dev server (mnohem rychlejší než Webpack)
- **@vitejs/plugin-react** (^6.0.1) - Plugin pro React v Vite
- **eslint** (^10.3.0) - Linter pro kód
- **eslint-plugin-react-hooks** - Pravidla pro React hooks
- **eslint-plugin-react-refresh** - Pravidla pro React refresh
- **@types/react** - TypeScript typy pro React (volitelně)
- **globals** - Globální proměnné pro ESLint

### 3. **index.html** - HTML template
- Vstupní bod aplikace pro browser
- Obsahuje `<div id="root"></div>` - cílový prvek pro React
- Načítá `main.jsx` jako modul

### 4. **src/main.jsx** - Entry point aplikace
- Inicializuje React aplikaci
- Používá `createRoot()` API (React 18+)
- Renduje `<App />` komponenta do elementu `#root`
- Používá `<StrictMode>` pro lepší vývojové chyby

### 5. **src/App.jsx** - Hlavní komponenta
- Funcional komponenta s React hooks
- Demonstrační komponenta s počítadlem (`useState`)
- Importuje logo Vite a React
- Příklad HMR (Hot Module Replacement) - live reload

### 6. **src/App.css** - Styly aplikace
- CSS styly pro App komponentu
- Příklad stylování React komponent

### 7. **src/index.css** - Globální styly
- Resetuje browser defaulty
- Definuje globální font a barevné schéma

### 8. **vite.config.js** - Konfigurace Vite
- Přidává React plugin pro JSX transformaci
- Nastavuje dev server (standardně localhost:5173)

### 9. **eslint.config.js** - Konfigurace linteru
- Nastavuje ESLint pravidla pro React kód
- Kontroluje kvalitu a konzistenci kódu
- Obsahuje flat config (nový formát ESLint)

### 10. **.gitignore** - Git ignorování
- `node_modules/` - Nikdy se commituje
- `dist/` - Build výstup
- `.env.local` - Místní environment proměnné

## Jak spustit vývojový server?

```bash
cd weather-app
npm install           # Nainstaluje všechny dependence
npm run dev          # Spustí dev server (obvykle na http://localhost:5173)
```

## Výhody této architektury

### Vite vs. Webpack
- ✅ **Bleskurychlé** - Native ES6 moduly při vývoji
- ✅ **Instant HMR** - Okamžitý reload při změně kódu
- ✅ **Menší konfigurace** - Zjednodušená setup
- ✅ **Rychlejší build** - Efektivní produkční balíčky

### React 19.x
- ✅ **Moderní API** - Nové features a optimalizace
- ✅ **Server Components** (experimentálně)
- ✅ **Lepší performance** - Optimalizované re-rendery

### ESLint konfiguraci
- ✅ **Detekce chyb** - Automatic problémů v kódu
- ✅ **React best practices** - Závěrečné pravidla pro hooks
- ✅ **Code consistency** - Jednotný styl kódu

## Příští kroky

1. **Vymazat demo obsah** - Přepsat App.jsx pro vlastní aplikaci

3. **Přidat styly** - Použít CSS moduly nebo CSS-in-JS
4. **API integrace** - Připojit backend API (pro weather app)
5. **Deployment** - Build a nasazení na hosting (Vercel, Netlify, atd.)

# Náhled buildu

## Poznámky

- Projekt je nyní připraven pro vývoj weather aplikace
- Všechny soubory mají detailní komentáře pro snadnější pochopení
- Vite automaticky refreshuje aplikaci při změně kódu (HMR)
- ESLint sleduje kvalitu kódu
- TypeScript lze snadno přidat později (přejmenovat .jsx na .tsx)
